MCP Discovery Service

The DNS for agent tools. A federated protocol for finding MCP servers by capability at runtime, with verifiable trust.

Why

MCP standardized how agents connect to tools — but not how they find them. Today, every MCP server is hard-configured in a JSON file before the agent starts:

{
  "mcpServers": {
    "filesystem": { "command": "npx", "args": ["@modelcontextprotocol/server-filesystem", "/tmp"] }
  }
}

This breaks when an agent needs a tool at runtime that wasn't pre-configured, when you have 14,000+ servers across 8 registries with no capability-based query, or when a server endpoint changes and every config needs updating.

MCP-DS adds a programmatic discovery layer — agent runtimes query for tools by capability, get back verified endpoints with trust scores, and connect dynamically.

Where It Fits in the Stack

Design at a Glance

🔍 Capability-Driven

Query by {domain, action, constraints}, not by server name. Agents ask "who can read files?" and get back matching servers with trust scores.

🌐 Federated

No single registry is the source of truth. Any registry can sync with any other. Clients query one, get results from many.

✅ Verifiable Trust

Every response carries a trust chain — namespace verification, security audits, publisher signatures. The client decides the threshold.

⚡ Runtime-Native

Agents discover tools dynamically, not at deploy time. The resolver caches results; the agent just says "I need X."

Protocol Components

ComponentWhat It DoesAnalogy
Registry API Standard REST API that any registry implements — query servers by capability, get trust chains DNS server
Capability Query Resolve {domain, action, inputs} → [MCP server URIs + trust] SRV record lookup
Server Card .well-known/mcp-server-card — discoverable metadata on any MCP host robots.txt
Trust Chain Signed attestations: registry signs identity, auditor signs security, publisher signs artifacts DNSSEC + CA chain
Federated Sync Zone-transfer API + peer query forwarding between registries DNS zone transfers

Quick Example

An agent needs a file-reading tool at runtime:

GET /v1/query?domain=storage&action=read_file&trust_min=0.7
{
  "results": [
    {
      "server": "io.github.mcp/filesystem",
      "uri": "mcp://filesystem.example.com",
      "version": "1.3.0",
      "trust": 0.92,
      "auth": ["oauth2"],
      "capabilities": ["read_file", "write_file", "list_files"]
    }
  ]
}

The runtime connects to the chosen server via standard MCP transport. No config files. No pre-declared endpoints.
